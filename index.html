GCD是好东西

gcd，即最大公约数，在数论题里有一席之地
（1）如果两个自然数是互质数，那么它们的最大公约数是1，最小公倍数是这两个数的乘积。
例如8和9，它们是互质数，所以（8，9）=1，[8，9]=72。
（2）如果两个自然数中，较大数是较小数的倍数，那么较小数就是这两个数的最大公约数，较大数就是这两个数的最小公倍数。
例如18与3，18÷3=6，所以（18，3）=3，[18，3]=18。
（3）两个整数分别除以它们的最大公约数，所得的商是互质数。
例如8和14分别除以它们的最大公约数2，所得的商分别为4和7，那么4和7是互质数。
（4）两个自然数的最大公约数与它们的最小公倍数的乘积等于这两个数的乘积。
例如12和16，（12，16）=4，[12，16]=48，有4×48=12×16，即（12，16）× [12，16]=12×16。
（5）a与b的最大公约数是最小的a与b的正线性组合,即对于方程xa+yb=c来说,若x,a,y,b都为整数,那么c的最小正根为gcd(a,b).

欧几里得算法

也即更相减损术通常用递归和模运算实现 

ll gcd(ll ,ll b){
    return !b?a:gcd(b,a%b);
}

一般来说这个就够用了，系统也自带了__gcd();来方便使用
而这个方法使用的是模运算，每次计算比较耗时，于是就有了下面的方法

Stein算法

Stein算法使用了加减和移位操作来减少耗时
（1）当a，b都是偶数，就分别除以二；
（2）当a为偶，b为奇，就将a除以二（反之亦然）；
（3）当a，b都为奇数，就将a=（a+b）/2，b=（a-b）/2；

然后再优化一下就成了：

ll gcd_core(ll a,ll b){
    if(!b)return a;
    if(!a)return b;
    while((!(a&0x1)))a>>=1;
    return a<b?gcd_core((b-a)>>1,a):gcd_core((a-b)>>1,b);
}
ll gcd_stein(ll a,ll b){
    ll c=0;
    while((!(a&0x1))&&(!(b&0x1)))a>>=1,b>>=1,c++;
    return (!a&0x1)?gcd_core(a>>1,b)<<c: gcd_core(b,a)<<c;
}

启发式合并

发表于 2022-11-16 分类于 信息学

这是玄学

那么什么是启发式合并呢？

我们假设有一列集合，我们需要按照某种规则将其合并，如果我们直接暴力的去合并，直接拿起两个集合进行枚举合并，那我们将得到一个n方的复杂度，这显然是坏的
我们考虑合并时是用一个集合向另一个集合合并，时间花费是当前集合的元素总数，那如果我们每次都用小的集合向大的集合合并，那我们就可以简化掉一些合并的时间花销
而这个简化会将nn的复杂度优化至nlogn
粗浅的意识流证明： 

当你在选取区间进行合并时，用小的集合向大的集合进行合并，这样做的话，
对于每次被合并的集合都存在一个集合比该集合的元素更多，这样的最有情况只需要合并n次，
即每次都选取新的集合向同一个集合合并，而最坏情况则是每次合并前后的集合大小相同，
这样就是n*log2(n)的复杂度，美滋滋

        scanf("%d",&k);
        if(k==1){

            if(x==y)continue;
            if(sz[ar[x]]>sz[ar[y]])swap(ar[x],ar[y]);
            x=ar[x],y=ar[y];
            for(int j=h[x];j;j=ne[j]){
                int k=to[j];
                if(a[k-1]==y)anss--;
                if(a[k+1]==y)anss--;                
            }
            for(int j=h[x];j;j=ne[j]){
                a[to[j]]=y;
                if(!ne[j]){
                    ne[j]=h[y];
                    h[y]=h[x];
                    sz[y]+=sz[x];
                    sz[x]=0;
                    h[x]=0;
                    break;
                }
            }
        }
        else{
            printf("%d\n",anss);
        }
    }
}

此处引入树上启发式合并
以每个节点为根子树都可以看作一个集合（看作二元组吧，key和value），我们向上贡献答案时候就可以采取合并的思路
根据最开始的思路，我们在选择合并时候，可以让小的子树向大的子树合并，但是这样有一个问题： 

我们在合并过程中怎么维护每个节点的子树中所有的颜色和出现次数？

面对题目的刁难，我们的选择是，不维护啊不维护
我们将子树规模最大的子节点成为重儿子，将其作为当前最大的集合与其他进行合并显然不会更劣，但是没法维护，那我们就把重儿子的子树染色状态保留，直接作为父母节点的当前已经计算过的子树，而轻儿子则直接遍历一边求出染色状态，然后清除它的贡献，只保留重儿子的。这样就很怪，但是很快。
时间复杂度看起来很玄学，一个节点如果在轻链上，他会被计算轻链的长度次 ，也即log(轻链长度)次（因为对于轻儿子来说，一定有一个比他大的重儿子存在），而作为一个重链上的节点，也只会被算1+log(重链前的轻链长度)次，均摊下来是个n*long(n)的复杂的，所以是极好的。 

#include<bits/stdc++.h>
using namespace std;
typedef long long ll;
const int N=2e5+10,M=1e5+10;
int n;
ll ans[M];
int a[M],ne[N],h[M],t[N],idx=2,cnt[M];
int sz[M],hav[M];
void dfs1(int x,int pa){
    sz[x]=1;
    for(int i=h[x];i;i=ne[i]){
        int y=t[i];
        if(y==pa)continue;
        dfs1(y,x);
        sz[x]+=sz[y];
        if(sz[hav[x]]<sz[y])hav[x]=y;
    }
}
ll ma,tmp;
void update(int x,int pa,int sign,int nop){
    cnt[a[x]]+=sign;
    if(cnt[a[x]]==ma)tmp+=a[x];
    if(cnt[a[x]]>ma)tmp=a[x],ma=cnt[a[x]];
    for(int i=h[x];i;i=ne[i]){
        int y=t[i];
        if(y!=nop&&y!=pa)
            update(y,x,sign,nop);
    }
}
void dfs2(int x,int pa,int op){
    for(int i=h[x];i;i=ne[i]){
        if(t[i]!=hav[x]&&t[i]!=pa)dfs2(t[i],x,0);
    };
    if(hav[x])dfs2(hav[x],x,1);
    update(x,pa,1,hav[x]);
    ans[x]=tmp;
    if(!op)update(x,pa,-1,0),ma=0,tmp=0;
}
int main(){
    scanf("%d",&n);
    for(int i=1;i<=n;i++){
        scanf("%d",&a[i]);
    }
    int u,v;
    for(int i=1;i<n;i++){
        scanf("%d%d",&u,&v);
        ne[idx]=h[u];
        t[idx]=v;
        h[u]=idx++;
        ne[idx]=h[v];
        t[idx]=u;
        h[v]=idx++;
    }
    dfs1(1,0);
    dfs2(1,0,1);
    for(int i=1;i<=n;i++)printf("%lld ",ans[i]);
    return 0;
}

A. Golden Spirit

讲河两岸各有n个老人，他们需要过桥玩X分钟，休息完再回来，你需要帮他们过桥，你自己过桥或者扶老人过桥均需要t分钟，给定一组n，x，t（1e9范围），求所需最短时间。

先把所有老人都扶到对岸显然是最优的，并且第一个到当前岸边的老人休息完后，即视为休息完毕
然后分为两种情况考虑：

1，把所有老人扶到对岸，然后等当前岸边第一个过来的老人（全部老人中第二个过桥的）休息完再走，此时的贡献是

2*2*n*t+x-(2*(n-1))*t

2,把所有老人扶到对岸，然后到对岸，等对岸第一个过来的老人（全部老人中第一个过桥的）休息完再走，此时的贡献是

2*2*n*t-(2*(n-1))*t+t

两个情况取max即可

代码实现：

#include<bits/stdc++.h>
using namespace std;
typedef long long ll;

int main(){
    int t;
    cin>>t;
    while(t--){
        ll n,x,k;
        scanf("%lld%lld%lld",&n,&x,&k);        
        ll ans1=(n*4*k)+max(0ll,x-(2ll*n*k-2*k));
        ll ans2=(n*4*k)+k+max(0ll,x-(2ll*n*k));
        cout<<min(ans1,ans2)<<"\n";
    }
    return 0;
}

C. Rencontre

讲一棵树，n个节点，读入u，v，w，表示u和v之间有双向边，距离为w，在树上有三次选项，第i次选择会等概率的从Mi个给定的点中选，然后找一个点，到三次选择的点距离和最小，求这个距离和的期望值

我们考虑算出三个点的简单路径，相加再除以二就是到三个的最小距离和的点的距离和，然后再算期望
直接算肯定不行，但是注意到本题是很多种可能求期望
我们可以利用换根dp求出每个点到其他所有点的距离和，这样可以求出在枚举任意三个点为根时的所有简单路径和，除以二便是所有到三次选择的点距离和最小的点的距离的总和
然后考虑哪些点可以给出贡献，我们只需要在树形dp计算路径时候，只记录被某次选择包含的点到根节点的路径（三次选择分别计算），然后换根求出所有点作为根时，三次选择的所有点分别到该根节点的的距离和。然后枚举三次选择，将维护好的另外两种选择的值直接累加过来，再除以对应选择的节点数算出期望即可

代码实现：

#include<bits/stdc++.h>
using namespace std;
const int N=2e5+1000,M=4e5+100;
typedef double db;
typedef long long ll;
int n,h[N],ne[M],t[M],val[M],idx=2;
int m[3],nu[N][3],uu[N][3];
ll d[N][3];
db anss;

void add(int u,int v,int w){
    ne[idx]=h[u];
    t[idx]=v;
    val[idx]=w;
    h[u]=idx++;
}

void dfs(int x,int pa){
    for(int i=h[x];i;i=ne[i]){
        if(i==(pa^1))continue;
        int y=t[i]; 
        dfs(y,i);
        for(int j=0;j<3;j++){
            nu[x][j]+=nu[y][j]+uu[y][j];
            d[x][j]+=(ll)(nu[y][j]+uu[y][j])*val[i]+d[y][j];
            
        }
    }
    return ;
}
void dp(int x,int fr,int e){
    if(fr!=0){
        for(int i=0;i<3;i++){
            d[x][i]=d[fr][i]+(1ll*m[i]-2ll*uu[x][i]-2ll*nu[x][i])*val[e];
        }

    }
    for(int i=h[x];i;i=ne[i]){
        int y=t[i];
        if(y==fr)continue;
        dp(y,x,i);
    }
    return ;
}
int main(){
    #ifdef LOCAL
    freopen("input.in","r",stdin);
    freopen("output.out","w",stdout);
    #endif
    scanf("%d",&n);
    int u,v,w;
    for(int i=1;i<n;i++){
        scanf("%d%d%d",&u,&v,&w);
        add(u,v,w);
        add(v,u,w);
    }
    
    int a;
    for(int i=0;i<3;i++){
        scanf("%d",&m[i]);
        for(int j=1;j<=m[i];j++){
            scanf("%d",&a);
            uu[a][i]=1;
        }
    }
    dfs(1,0);
    dp(1,0,0);
    for(int k=0;k<3;k++){
        for(int j=k+1;j<3;j++){
            for(int i=1;i<=n;i++){
                if(uu[i][k])anss+=(db)d[i][j]/(1.0*m[j]*m[k]);
            }
        }
    }
    printf("%.10lf",anss/2);
    return 0;
}

H Message Bomb

讲有n个聊天群，m个人，s次操作，s行读入t，x，y，操作分三种t=1 第x人加入y组，t=2，第x人退出y组，t=3 第x人在y组发一条信息，此时在组内所有人都收到这条信息，求最终每个人收到多少信息

我们把结束时候看作所有人都出群了，我们用该人出群时间群内信息减去他入群时候信息数和他在这个群发的消息数便是他这次进群收到的所有信息数
对于每次操作，维护每个群内当前的信息总数
维护n个数据结构（我用的map）代表第i个组内现在有哪些人，以及他入群时候的群内的信息数，该人入过发一条消息，视作他入群时的信息多一条即可，然后在退群时候记录答案

代码实现

#include<bits/stdc++.h>
using namespace std;
const int N=2e5+1000,M=4e5+100;
typedef double db;
typedef long long ll;

int mes[N],n,m,s,anss[N];
map< int,int >mp[N];

int main(){
    #ifdef LOCAL
    freopen("input.in","r",stdin);
    freopen("output.out","w",stdout);
    #endif
    scanf("%d%d%d",&n,&m,&s);
    int t,x,y;
    for(int i=1;i<=n;i++)mes[i]=1;
    for(int i=1;i<=s;i++){
        scanf("%d%d%d",&t,&x,&y);
        if(t==1){
            mp[y][x]=mes[y];
        }
        else if(t==2){
            anss[x]+=mes[y]-mp[y][x];
            mp[y][x]=0;
        }
        else if(t==3){
            mes[y]++;
            mp[y][x]++;
        }
    }
    for(int i=1;i<=n;i++){
        for(auto x:mp[i]){
            if(x.second !=0){
                anss[x.first]+=mes[i]-x.second;
            }
        }
    }
    for(int i=1;i<=m;i++){
        printf("%d\n",anss[i]);
    }
    return 0;
}
